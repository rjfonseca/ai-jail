# v0.7.0

## Fix: Crush now runs on the real terminal instead of the PTY proxy

Crush 0.51.x was rendering incorrectly inside `ai-jail`'s status-bar/PTY path: panes painted over old content, sidebars overlapped the main view, and the screen accumulated stale fragments across redraws.

The root cause is that Crush expects a richer terminal contract than the PTY proxy currently emulates. It probes terminal capabilities directly and uses rendering behavior that does not round-trip cleanly through the `vt100` proxy model.

`ai-jail crush` now detects that command and skips the PTY/status-bar proxy entirely, launching Crush on the real terminal instead. This restores correct rendering at the cost of hiding the persistent `ai-jail` footer for Crush sessions only.

### Related terminal-state fix

The status bar no longer uses `DECSC` / `DECRC` (`ESC 7` / `ESC 8`) to save and restore the cursor. Instead it restores cursor and drawing attributes from the tracked `vt100` screen state, avoiding conflicts with TUIs that use the terminal save slot themselves.

### Tests

- Added tests covering the Crush direct-TTY fallback command detection.
- Added a status-bar test ensuring terminal-state restoration no longer relies on `ESC 7` / `ESC 8`.

### Maintenance

- Updated the crate from Rust 2021 edition to Rust 2024 edition.
